从长远来看：
可以很清晰地看到文件的修改记录
便于比对新旧内容的异同

手动提交更加灵活
每次提交具有明确的意义
便于对各个提交的关键点回顾
但是比较麻烦，需要自己手动操作

自动保存比较方便，无需人为去操作
但是保存的版本可能不具有明确的意义
不便于理解每次的提交

一次提交中保存多个文件更具有整体性
提交的整体结构更加完整清晰

git log --stat 显示提交的历史记录，包括每次提交哪些文件修改了
git log 仅仅显示提交的历史记录
git diff 前一次的commit id 这一次的commit id

git checkout commit id 可以将本地仓库的文件切换到任意一次提交上去
所以如果本次提交了破坏的更改，那么可以通过上述checkout到破坏之前的一次提交上去
